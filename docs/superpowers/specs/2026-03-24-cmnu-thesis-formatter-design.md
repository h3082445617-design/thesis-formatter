# 中央民族大学学位论文排版工具设计文档

**日期**: 2026-03-24
**优先级**: 紧急
**项目**: 为生物多样性保护资金机制硕士论文做最终排版

---

## 1. 项目概述

### 背景
用户拥有一份接近完成的硕士学位论文（《昆蒙框架下生物多样性保护资金机制与ABS机制研究》），需要按照**中央民族大学生命与环境科学学院**的严格排版规范进行最终格式化。

### 目标
创建一个 Python 脚本，自动将用户的论文按规范格式化，支持所有部分（摘要、目录、正文、参考文献、附录、致谢）。

### 范围
- ✅ 不处理：封面、作者声明（用户自己处理）
- ✅ 处理：摘要、目录、正文、参考文献、附录、致谢、页码

---

## 2. 核心排版规范

### 2.1 页面设置
- **纸张**: A4 白色，双面打印
- **边距**: 上下 2.5cm，左右 2.2cm，装订线 0.5cm
- **页码**:
  - 前置部分（摘要、目录）: 大写罗马数字 (I, II, III...)
  - 正文（第1章起）: 阿拉伯数字 (1, 2, 3...)
  - 位置: 页面底端居中

### 2.2 摘要部分

#### 中文摘要
| 元素 | 格式 |
|------|------|
| 标题"摘要" | 三号黑体，居中，字间空一格，单倍行距，段前24磅/段后18磅 |
| 正文 | 小四号宋体，1.5倍行距，首行缩进2字符，两端对齐 |
| "关键词:" | 四号黑体，左对齐 |
| 关键词内容 | 小四号宋体，1.5倍行距，3-5个词，分号隔开，最后不打标点 |

#### 英文摘要 (ABSTRACT)
| 元素 | 格式 |
|------|------|
| 标题 | 三号 Times New Roman 加黑，居中，单倍行距，段前24磅/段后18磅 |
| 正文 | 小四号 TNR，1.5倍行距，首行缩进2字符，两端对齐 |
| "KEY WORDS:" | 四号 TNR 加黑 |
| 关键词 | 小四号 TNR，1.5倍行距，冒号后空两字符开始 |

### 2.3 目录
| 元素 | 格式 |
|------|------|
| 标题"目录" | 三号黑体，居中，字间空一格，单倍行距，段前24磅/段后18磅 |
| 章标题 | 小四号宋体 |
| 节标题 | 小四号 TNR |
| 行距 | 1.5倍，标题与页码用"……"连接 |

### 2.4 正文格式

#### 全局设置
- **字体**: 小四号宋体（数字/英文/罗马字用 TNR，希腊文用 Symbol）
- **行距**: 1.5倍
- **缩进**: 首行缩进2字符，两端对齐
- **标点**: 半角

#### 标题层级
| 等级 | 格式 |
|------|------|
| 一级（章） | 三号黑体，居中，单倍行距，段前24磅/段后18磅 |
| 二级（1.1）| 四号黑体，左对齐，固定行距20磅，段前24磅/段后6磅 |
| 三级（1.1.1）| 小四号黑体，左对齐，固定行距20磅，段前12磅/段后6磅 |

### 2.5 图表与公式

#### 图
- 编号: 按章编号（如"图2-5"）
- 位置: 图片下方，居中
- 格式: 五号楷体_GB2312，1.25倍行距
- 图序与图题间空1字距

#### 表
- 编号: 按章编号
- 位置: 表格上方，居中
- 格式: 同图（五号楷体_GB2312，1.25倍行距）
- 表内文字: 小四号宋体，1.5倍行距（未明确，按正文处理）
- 跨页表: 续表上方注明"续表"并重复表头

#### 公式
- 格式: 另起一行
- 编号: 按章编排，括号括起，右边行末

### 2.6 参考文献
| 元素 | 格式 |
|------|------|
| 标题"参考文献" | 三号黑体，居中，单倍行距，段前24磅/段后18磅 |
| 序号 | [1]、[2]等方括号形式 |
| 正文 | 五号宋体，1.5倍行距，两端对齐 |
| 缩进 | 首行无缩进，悬挂缩进（第二行起对齐） |
| 作者 | 超3位只列前3位，后加"，等"或"，et al." |

### 2.7 附录与致谢
| 元素 | 格式 |
|------|------|
| 标题 | 三号黑体，居中，单倍行距，段前24磅/段后18磅 |
| 正文 | 小四号宋体，1.5倍行距，首行缩进2字符，两端对齐 |

## 3. 实现方案

### 方案 C（混合方案）- 推荐

#### 第一阶段：紧急脚本（1-2天）
**目标**: 快速处理用户的论文

**输出物**:
- `src/formatters/cmnu_formatter.py` - 中央民族大学格式化器
- `scripts/format_thesis.py` - 命令行脚本
- 用户论文的排版版本：`论文_formatted.docx`

**工作流程**:
```
输入论文 → 检测各部分 → 应用格式规则 → 处理页码分节 → 输出结果
```

**关键实现**:
1. **页码分节**: 检测正文开始位置，插入分节符，应用不同页码格式
2. **格式检测**: 识别各部分（摘要、目录、正文、参考文献、附录、致谢）
3. **格式应用**: 逐部分应用对应的字体、行距、缩进、对齐规则
4. **字体混用**: 保留原有中英文，应用正确的字体（宋体vs TNR）
5. **特殊处理**: 图表编号、目录生成、参考文献悬挂缩进

#### 第二阶段：集成到 Web 工具（后续）
- 复用格式化器逻辑
- 在 Flask 工具中添加"中央民族大学"标准选项
- 用户可通过 Web 界面上传论文

### 4.2 CmnuFormatter 类设计

```python
class CmnuFormatter:
    """中央民族大学排版格式化器"""

    def format_document(self, input_path):
        """主入口: 格式化论文"""
        doc = Document(input_path)

        # 1. 检测各部分
        parts = self._detect_parts(doc)

        # 2. 应用格式
        self._format_abstract(doc, parts)
        self._format_toc(doc, parts)
        self._format_body(doc, parts)
        self._format_references(doc, parts)
        self._format_appendix(doc, parts)

        # 3. 处理页码（关键）
        self._setup_page_numbers(doc, parts)

        # 4. 保存
        output_path = input_path.replace('.docx', '_formatted.docx')
        doc.save(output_path)
        return output_path
```

### 4.3 关键函数

| 函数 | 职责 | 复杂度 |
|------|------|--------|
| `_detect_parts()` | 识别摘要、目录、正文等各部分的段落范围 | 高 |
| `_setup_page_numbers()` | 实现前置罗马数字+正文阿拉伯数字的分节页码 | 高 |
| `_format_body()` | 应用标题层级、字体、行距等 | 中 |
| `_format_abstract()` | 摘要特殊格式 | 中 |
| `_format_references()` | 参考文献悬挂缩进 | 中 |

### 4.4 部分检测策略（关键）

规范未明确说明脚注/尾注格式，采用如下处理：

- **标记为"待手动处理"**，提示用户检查

## 7. 错误处理与用户交互

### 自动处理失败的降级方案

| 场景 | 优先处理 | 失败时降级 | 最终方案 |
|------|---------|-----------|---------|
| 部分边界检测失败 | 关键词匹配 | 启发式检测 | 用户手动指定 |
| 页码分节失败 | OXml操作 | 简化分节 | 全部用阿拉伯数字 |
| 字体不可用 | 系统字体 | 降级到通用字体 | 记录警告，继续 |
| 格式应用异常 | 逐部分应用 | 跳过该部分 | 记录日志 |

### 用户交互模式

```bash
# 完全自动模式（推荐）
python scripts/format_thesis.py input.docx

# 指定关键部分（当自动检测失败时）
python scripts/format_thesis.py input.docx \
  --abstract-start 3 --abstract-end 8 \
  --body-start 15 --body-end 150 \
  --references-start 151 --references-end 200

# 调试模式（显示检测结果）
python scripts/format_thesis.py input.docx --debug
```

## 8. 错误处理

| 场景 | 处理方式 |
|------|---------|
| 无法检测摘要部分 | 提示用户，允许手动指定 (参数或交互) |
| 页码分节失败 | 回退到单一页码格式，提示用户 |
| 字体不可用 | 自动降级到系统默认字体，记录警告 |
| 格式应用失败 | 记录日志，继续处理其他部分，汇总报告 |
| 表格/脚注异常 | 标记为"待手动检查"，输出提示 |
